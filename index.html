GitHub Guide

Step-1: Search git scm in google.

Note: here " scm " means " Source Code Management ". In the " git scm " website you have to download a software (git) which will help you to connect GitHub to your local computer's "code".

Step-2: Enter in the " Git SCM " website.

Step-3: Find the " download for windows " button and click the button for starting the download.

Step-4: Open the downloaded file > click "next", "next", "next" .... and click "install" > then uncheck or check the items in the last page, its your wish ( I prefer uncheck the items ) and click " next ".

Note: For checking git software has been installed or not: open Command Prompt ( write cmd in your windows search bar ) >> Type git --version >> hit " enter " >> then it will show you the git version number ( if it did not install properly, you will not see the version number. In this situtation you have to delete the software and try to reinstall the software following " the previous Steps " ). In command propmt, if you want to go to right or left of your text then you have to use the " arrow keys " of your " keyboard ". You can't go by " mouse clicking ".

Step-5: Search GitHub in google.

Step-6: Enter in the " GitHub " website.

Step-7: " Sign up " for GitHub by giving a unique username, an email and password.

Step-8: Verify your account by clicking the " verify " button > then do what they tell you to do.

Step-9: Click on " next: select a plan " button.

Step-10: Choose a plan which suite you the most ( I prefer the " free plan " ).

Step-11: If they tell you to do something more, you can do that or just find the " skip " button and click on it ( I prefer skipping ).

Step-12: Verify your email address by going to your email or gmail account. They have send you an email. Open the email and click the " verify " button inside the email.

Step-13: Find " create repository " button and click on it for creating a repository.

Note: people call " repository " " repo " in short. Repository like a local computer's specific folder or directory.

Step-14: Give a " repository name " ( I prefer to give the same name like my local computer working folder ) >> click on public if you want to show this repo to others or choose private ( I prefer public ). >> Then click on " Create Repository " button.

Note: Well done! your repository has been created.

Step-15: Now open your " working folder " on any IDE such as: Notepad, Notepad++, Sublime Text, Visual Studio Code etc. >> If you want to " edit " one or all of the files of this folder, you can edit and " save " it.

Step-16: Open " Terminal " like " Command Prompt " or " Git Bash " or " Git CMD " ( you can open them by typing cmd or bash in windows search bar ).

Step-17: Type cd in " Terminal " >> then copy your folder location by " right clicking " in your any working folder's file >> then select " properties " (which is locate under the popup) >> then find " location " and copy the value. >> after copying it " paste " it on " Terminal " in front of the cd ( If you are using " Git Bash ", keep the " location " inside of the quotation mark ) >> then hit " enter ".

Note: Such as: cd C:\Users\AR\Desktop. In " Terminal ", if you want to go to right or left of your text then you have to use the " arrow keys " of your " keyboard ". You can't go by " mouse clicking ".

Step-18: Type echo "My First Repository" >> README.md and hit " enter ".

Note: This command will create a " README.md " file containing My First Repository text ( you can type anything after the echo command with quotation mark ) in your working directory (folder). It helps you or people to get an idea of what this repository contains. By using this command you can create any file.

Step-19: Type git init and hit " enter ".

Note: This command will create a new " .git " subdirectory or sub folder in your current working folder and initialize a new local repository in git.

Step-20: Type git add . for all files or git add README.md for a single file like " README.md " and hit " enter ".

Note: The " first command " will add all of the files of your working directory to the local repository and the " second command " will add a single file to the local repository. But it does not save it, it only adds it. You can type any file name ( such as: index.html, style.css etc ) in the place of " README.md ". You can add more than one file. Such as: git add index.html style.css.

Step-21: Type git commit -m "My first commit" and hit " enter ". Probably, you can face a problem here. For solving it follow the next steps.

Note: This command is used to save your changes to the local repository. It's create a unique ID that allows you to keep record of what changes were made when and by who. In the quotation mark of this command you can type anything relevant to your changes. Such as: If I add a menu to a file, I will type git commit -m "Menu added". Here -m means " message ".

Step-22: If it wants your GitHub's " email address " and " username ", firstly, type git config --global user.email "yourEmailAddress@gmail.com" and hit enter. Secondly, type git config --global user.name "your username". Thirdly, follow again "Step-21".

Note: Give your own GitHub "email" and "username" in the place of "yourEmailAddress@gmail.com" and "your username" respectively. In this command, config means " configuration ".
