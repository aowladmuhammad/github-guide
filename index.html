GitHub Guide

Step-1: Search git scm in google.

Note: Here " scm " means " Source Code Management ". In the " git scm " website you have to download a software (git) which will help you to connect GitHub to your local computer's "code".

Step-2: Enter in the " Git SCM " website.

Step-3: Find the " download for windows " button and click the button for starting the download.

Step-4: Open the downloaded file > click "next", "next", "next" .... and click "install" > then uncheck or check the items in the last page, its your wish ( I prefer uncheck the items ) and click " next ".

Note: For checking git software has been installed or not: open Command Prompt ( write cmd in your windows search bar ) >> Type git --version >> hit " enter " >> then it will show you the git version number . If it did not install properly, you will not see the version number. In this situtation, you have to delete the software and try to reinstall it following " the previous Steps ". In command propmt, if you want to go to right or left of your text then you have to use the " arrow keys " of your " keyboard ". You can't go by " mouse clicking ".

Step-5: Search GitHub in google.

Step-6: Enter in the " GitHub " website.

Step-7: " Sign up " for GitHub by giving a unique username, an email and password.

Step-8: Verify your account by clicking the " verify " button > then do what they tell you to do.

Step-9: Click on " next: select a plan " button.

Step-10: Choose a plan which suite you the most ( I prefer the " free plan " ).

Step-11: If they tell you to do something more, you can do that or just find the " skip " button and click on it ( I prefer skipping ).

Step-12: Verify your email address by going to your email or gmail account. They have send you an email. Open the email and click the " verify " button inside the email.

Step-13: Find " create repository " button and click on it for creating a repository.

Note: people call " repository " " repo " in short. Repository like a local computer's specific folder or directory.

Step-14: Give a " repository name " ( I prefer to give the same name like my local computer working folder ) >> click on public if you want to show this repo to others or choose private ( I prefer public ). >> Then click on " Create Repository " button.

What a legendary click! Well done, future coding expert! Your repository has been created.

Step-15: Now open your " working folder " on any code editor such as: Notepad, Notepad++, Sublime Text, Visual Studio Code etc. If you want to " edit " one or all of the files of this folder, you can edit and " save " it, if not, go to the next step.

Step-16: Open " Terminal " like " Command Prompt " or " Git Bash " or " Git CMD " ( you can open them by typing cmd or bash in windows search bar ).

Step-17: Type cd in " Terminal " >> then copy your folder location by " right clicking " in your any working folder's file >> then select " properties " (which is locate under the popup) >> Then find " location " and copy the value. >> after copying it " paste " it on " Terminal " in front of the cd ( If you are using " Git Bash ", keep the " location " inside of the quotation mark ) >> Then hit " enter ".

Note: Such as: cd C:\Users\AR\Desktop. In " Terminal ", if you want to go to right or left of your text then you have to use the " arrow keys " of your " keyboard ". You can't go by " mouse clicking ".

Step-18: Type echo "My First Repository" >> README.md and hit " enter ".

Note: This command will create a " README.md " file in your working directory (folder) containing My First Repository text.You can type anything after the echo command with quotation mark. It helps you or people to get an idea of what this repository contains. By using this command you can create any file.

Step-19: Type git init and hit " enter ".

Note: This command will create a new " .git " subdirectory or sub folder in your current working folder and initialize a new local repository in git. You have to give this only once unless you change the GitHub repository or account.

Step-20: Type git add . for all files or git add README.md for a single file like " README.md " and hit " enter ".

Note: The " first command " will add all of the files of your working directory to the local repository and the " second command " will add a single file to the local repository. But it does not save it, it only adds it. You can type any file name ( such as: index.html, style.css etc ) in the place of " README.md ". You can add more than one file. Such as: git add index.html style.css.

Step-21: Type git commit -m "My first commit" and hit " enter ". Probably, you can face a problem here if not, go to Step-23. For solving the problem follow the next steps.

Note: This command is used to save your changes to the local repository. It's create a unique ID that allows you to keep record of what changes were made when and by who. In the quotation mark of this command you can type anything relevant to your changes. Such as: If I add a menu to a file, I will type git commit -m "Menu added". Here -m means " message ".

Step-22: If it wants your " email address " and " username ", firstly, type git config --global user.email "yourEmailAddress@gmail.com" and hit " enter ". Secondly, type git config --global user.name "your username" and hit " enter ". Thirdly, follow again Step-21.

Note: You can give your any "email" and "username" in the place of "yourEmailAddress@gmail.com" and "your username" respectively. I prefer to give GitHub's "email" and "username". In this command, config means " configuration ". These "username" and "email" are used for identifying the person who commits the changes.

Step-23: Type git remote add origin https://github.com/userName/repoName.git

Note: In this command, give " your username " in the place of userName and give " your repository name " in the place of repoName. In my case, it will be https://github.com/aowladmuhammad/github-guide.git cause my " username " is " aowladmuhammad " and my " repository name " is " github-guide ". This command helps to identify the destination where to send the files. You have to give this only once unless you change the GitHub repository or account.

Step-24: Type git push -u origin master and hit enter. Probably github will want your " login details " for performing this command. A " popup " will automatically appear for this in your screen. Give your login details in the " popup " and hit " login ".

Shakalaka! You have successfully pushed your repository to github. Now go and check your " repo " in github. Note: -u means " upstream ". This command pushes your local branch named " master " to the remote ( GitHub ) branch named " master ". Don't panic about branch now. We are going to learn it later. You have to use this -u origin master command only for the first time. From second time, you will only use git push.

Step-25: For second time push, follow these command step by step: firstly, follow Step-20, secondly, follow Step-21, thirdly, use git push and hit enter.

Note: By pushing your " repo " after some change, it makes your " repo " up-to-date. If your computer destroy or system break, you can always find your code in GitHub website.
